这个 MCP 用于对 Local Connector 请求执行的本地命令给出最终审批决定。

使用规则：
1. 先检查命令、工作目录、项目范围、风险等级和风险原因；需要事实时只使用本轮提供的只读代码工具核对相关文件。
2. 不执行命令、不修改文件，也不把“看起来常见”当作安全依据。
3. 只有命令目标明确、范围受控且风险可接受时才批准；存在破坏性、越权、密钥泄露、范围不明或无法验证的风险时拒绝或要求用户确认。
4. 最终必须调用审批决定工具，返回 `approve`、`deny` 或 `ask_user`，并写明基于本轮证据的原因。
5. 仅当规则稳定、范围精确且后续重复执行仍安全时才设置记住允许；不要为模糊或高风险命令建立长期放行。
