# 2.0.5 安全加固实施方案

## 背景

2026-07-13 对当前分支进行静态审查后，发现若干需要在 2.0.5 收敛的安全和工程质量问题：

- Docker 默认 `local` 环境会带开发默认凭据和默认内部 secret 启动，且多个服务端口默认发布到所有网卡。
- Task Runner 远程 SSH 的 `accept_new` 主机密钥策略会在指纹不匹配时覆盖 `known_hosts`。
- 多个服务仍允许长期 Bearer token 放在 URL query 中。
- user service 登录接口缺少可见的限速、失败计数或临时锁定。
- 源码体量门禁是名单制，实际已有多个新增超 700 行/40KB 的热点文件没有被 CI 拦截。
- 仓库直接跟踪多平台 `rg` 二进制，并存在重复副本，增加仓库体积和供应链维护成本。

## 目标

1. 默认部署不再把带弱默认凭据的服务暴露到局域网。
2. SSH 主机密钥策略符合常见语义：`accept_new` 仅信任首次出现的主机，指纹变化必须失败。
3. URL query token 逐步退出，只保留短期、单用途 ticket 类场景。
4. 登录入口具备基础抗爆破能力。
5. CI 对新增大代码文件和大文件有可执行的强制门禁。
6. 大二进制来源和重复副本有明确治理路径。

## 分阶段实施

### 第一阶段：低破坏安全修复

- 将 `docker/compose.yml` 中业务服务、前端和 Harness 的端口默认绑定到 `127.0.0.1`。
- 新增统一环境变量 `CHATOS_PUBLIC_BIND_HOST`，需要外部访问时显式设置为 `0.0.0.0` 或指定网卡地址。
- 保留 Consul、MongoDB、Sandbox Manager 现有的本机绑定默认值。
- 修改 SSH host key policy：
  - `Match`：允许连接。
  - `NotFound + accept_new`：写入 `known_hosts` 后允许连接。
  - `Mismatch`：始终失败，不允许 `accept_new` 覆盖。
  - `strict + NotFound`：失败。
- 为 SSH 策略增加单元测试，锁定 mismatch 行为。

### 第二阶段：URL query token 迁移

- 普通 REST 鉴权只接受 `Authorization: Bearer ...`。
- SSE/WebSocket 类浏览器限制场景改为短 TTL、单用途 ticket。
- 对仍需兼容的旧入口增加开关，默认关闭，并在日志中记录弃用告警。
- 更新前端 `EventSource` 连接方式：优先使用支持 header 的 fetch streaming/polyfill；如果必须使用原生 `EventSource`，先换取一次性 ticket，而不是长期 token。

### 第三阶段：登录保护

- 在 user service 登录接口增加账号和 IP 维度的失败计数。
- 对连续失败触发指数退避或短时间锁定。
- 失败响应保持统一，避免用户名枚举。
- 增加配置项控制窗口长度、最大失败次数和锁定时间。

### 第四阶段：代码体量和大文件门禁

- 为 `scripts/code-size-report.sh` 增加 `--fail` 模式，超过阈值直接非零退出。
- 在 CI 中执行全仓阈值门禁，而不是只检查手工维护的热点名单。
- 将当前超阈值文件列入拆分计划：
  - `local_connector_client/core/src/skills/native/artifacts.rs`
  - `local_connector_client/core/src/skills/mod.rs`
  - `local_connector_client/frontend/electron/main.cjs`
  - 已完成拆分：`plugin_management_service/backend/src/seed.rs`
  - 已完成拆分：`plugin_management_service/backend/src/store.rs`
  - 已完成拆分：`task_runner_service/backend/src/services/plugin_management_policy.rs`
  - 已完成拆分：`task_runner_service/backend/src/services/skill_runtime.rs`
  - 已完成拆分：`local_connector_service/backend/src/store/mongo.rs`
  - 已完成拆分：`local_connector_client/frontend/src/api.ts`
- 对 `bundled-tools` 中的多平台 `rg` 二进制建立来源、版本、hash 记录，并消除重复副本；后续优先迁移到构建/发布阶段下载并校验。

## 验收标准

- `docker compose -f docker/compose.yml config` 能正常生成配置，默认端口映射包含 `127.0.0.1`。
- SSH `accept_new` 对 `Mismatch` 返回错误，对 `NotFound` 仍允许首次记录。
- 新增测试能覆盖 SSH mismatch 不自动覆盖。
- URL query token 迁移完成后，服务端默认不接受长期 token query 参数。
- 登录失败保护有单元测试或集成测试覆盖。
- CI 能对新增超阈值源码文件失败。

## 当前 2.0.5 已完成范围

已落地以下风险高、兼容影响可控的修复：

- Docker 默认端口绑定收敛到本机。
- SSH `accept_new` 不再接受指纹变化。
- 补充 SSH 策略测试。
- `scripts/code-size-report.sh` 增加 `--fail` 模式。
- 新增 `scripts/code-size-baseline.tsv`，当前历史热点只能保持或下降，新增热点会使 CI 失败。
- GitHub Actions 与 Drone 接入源码体量门禁。
- GitHub Actions 与 Drone 接入大文件门禁，阻断未批准的新增大文件。
- 删除 `local_connector_client/frontend/electron/resources/bundled-tools/windows-x64/rg.exe` 重复副本，Electron resources 由 Windows 打包脚本从根目录 `bundled-tools` 生成。
- 新增 `bundled-tools/SHA256SUMS` 与 `scripts/check-bundled-tools-integrity.sh`，CI 校验 bundled `rg` 二进制哈希和清单完整性。
- 拆分 `plugin_management_service/backend/src/seed.rs`：将内部 Skill catalog/hash/include 逻辑迁移到 `seed/internal_skills.rs`，该文件已从源码热点 baseline 中移除。
- 拆分 `plugin_management_service/backend/src/store.rs`：将 MongoDB 索引初始化迁移到 `store/indexes.rs`，该文件已从源码热点 baseline 中移除。
- 拆分 `task_runner_service/backend/src/services/plugin_management_policy.rs`：将策略单元测试迁移到 `plugin_management_policy/tests.rs`，运行时代码文件已从源码热点 baseline 中移除。
- 拆分 `task_runner_service/backend/src/services/skill_runtime.rs`：将 Skill runtime 单元测试迁移到 `skill_runtime/tests.rs`，运行时代码文件已从源码热点 baseline 中移除。
- 拆分 `local_connector_service/backend/src/store/mongo.rs`：将 MongoDB 索引初始化迁移到 `store/mongo/indexes.rs`，该文件已从源码热点 baseline 中移除。
- 拆分 `local_connector_client/frontend/src/api.ts`：将 API 类型与 transport 迁移到 `apiTypes.ts`、`apiTransport.ts`，聚合 API 文件已从源码热点 baseline 中移除。
- user service 用户名/密码登录增加进程内用户名与来源 IP 维度失败计数和临时锁定，禁用账号也返回统一失败响应，避免用户名/账号状态枚举。
- 补充登录节流单元测试。
- Task Runner SSE 改为 `POST /api/auth/sse-ticket` 换取 60 秒一次性 ticket，`EventSource` 只携带 `sse_ticket`。
- Task Runner 通用鉴权拒绝旧的 `access_token` / `token` query 参数。
- Project Management 与 Plugin Management 普通 REST 鉴权拒绝旧的 `access_token` / `token` query 参数。
- Local Connector 普通受保护接口拒绝旧的 `access_token` / `token` query 参数；设备 connect WebSocket 仅在显式打开兼容开关时允许 query token，且 `Authorization` header 始终优先。

登录节流当前是单实例进程内保护；多实例部署需要在后续迁移到 MongoDB、Redis 或网关级限流。Local Connector 的设备 connect WebSocket 仍保留显式兼容开关，后续需要在确认旧配对客户端迁移完成后完全移除 query token 兼容入口。
