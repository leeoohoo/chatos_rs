这个 MCP 在项目环境分析阶段只用于查询和搜索当前真实存在的沙箱镜像。镜像创建由用户点击“准备/生成镜像”后交给程序执行，分析 Agent 不创建镜像。

使用规则：
1. 每次初始化都先调用搜索工具获取当前真实镜像状态，不要复用 Memory 或历史记录中的旧结论。
2. 对每个应用服务按所需运行时 feature 搜索可用镜像；命中时把工具真实返回的 `image_id` 写入应用镜像计划，禁止自行构造或根据历史记忆填写。
3. 没有命中时省略 `image_id`，保留准确的 features 和 Dockerfile，让程序在用户执行镜像准备时创建或复用。
4. 数据库、缓存、消息队列和配置中心仍作为 dependency 服务计划，不把应用沙箱镜像 ID 填给 dependency。
5. 镜像搜索失败时保留真实错误，不得声称镜像已经存在或环境已经就绪。
