# IM Service

## 模块概述

IM Service 是 Agent Stack 的消息投递层。

它让整个平台具备“像 IM 一样协作”的基础能力：用户消息先进入消息服务，后台处理完成后再把结果主动推送回来，而不是把前端直接绑定到原始模型流式输出。

## 这个服务负责什么

- IM 视角下的消息持久化
- 在线客户端的 WebSocket 主动推送
- 用户与联系人交互的投递语义
- 接收后端处理结果并回推到用户侧界面

## 为什么需要独立 IM 服务

如果把消息交互和后台执行完全绑死在一起，前端体验就会退化成“等模型流式吐字”。

独立 IM 服务的价值在于：
- 把“消息交互”和“后台执行”解耦
- 支持任务执行完成后的异步回复
- 保留面向用户可见的消息状态与投递链路
- 为后续用户管理、消息归档和多端同步预留清晰边界

## 目录结构

- `backend/`：Rust IM 服务

## 后端快速启动

```bash
cd backend
cargo run --bin im_service
```
